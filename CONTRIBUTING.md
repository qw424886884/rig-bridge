# Contributing

Issues and pull requests are welcome after the first public release.

## Before Opening an Issue

- Use the latest published version of Blender and Rig Bridge.
- Confirm that both armatures are humanoid and that the source has an active Action.
- Remove private character data and customer names from screenshots and sample files.
- Describe the source rig family, target rig family, rest pose, Blender version, and whether root motion is expected.

## Pull Requests

- Keep changes focused on one behavior.
- Do not include proprietary rigs, motions, add-ons, logos, or copied source code.
- Add a reproducible test or fixture description for recognition and retargeting changes.
- Validate the manifest and release package with Blender's extension CLI.
- Preserve the independent `Remap` sidebar category.
